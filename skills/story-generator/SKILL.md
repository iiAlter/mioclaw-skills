# Story Generator - 故事生成技能

根据用户需求生成原创故事，可用于后续分镜制作。

## 输入（用户提供）

- **主题/关键词**：想讲什么故事（或者让 AI 自由发挥）
- **角色**：使用现有角色（小白、小鸡毛）或新角色
- **风格**：温情治愈 / 幽默搞笑 / 冒险探索 / 友情 / 亲情 / 睡前温馨
- **长度**：短篇（3-4格）/ 中篇（5-6格）/ 长篇（8-9格）

## 用户未指定时

- 角色：默认使用小白和小鸡毛（两只小狗）
- 风格：默认温情治愈
- 长度：根据情节复杂度判断

## 输出格式

```markdown
## 故事主题选项

1. **《主题A》**：一句话介绍
2. **《主题B》**：一句话介绍
3. **《主题C》**：一句话介绍

---

请选择一个主题（回复数字或标题），或者告诉我你自己的想法。
```

**用户选择后**：

```markdown
## 《故事标题》

[故事正文]

---

**故事结构分析**：
- 起：
- 承：
- 转：
- 合：

**建议格数**：X 格

**背景音乐建议**：
- 音乐风格：
- 推荐曲目：
```

## 故事要求

1. **角色一致**：保持角色性格一致
   - 小白：白色小狗，温柔细心，有时有点敏感
   - 小鸡毛：金黄色小狗，活泼开朗，有时有点冲动

2. **情节健康**：温馨、积极、温暖

3. **避免**：过于复杂的情节、过多的新角色、说教内容

4. **角色数量**：尽量少引入新角色（最多1-2个），以原有角色为主

## 风格说明

| 风格 | 特点 |
|------|------|
| 温情治愈 | 温暖、感动、暖心结尾 |
| 幽默搞笑 | 逗趣、意外、反转 |
| 冒险探索 | 刺激、挑战、成长 |
| 友情 | 朋友陪伴、误会→和好 |
| 亲情 | 家人关爱、分离→团聚 |
| 睡前温馨 | 平静、柔和、美满结局 |

## 示例

**用户输入**："生成一个下雨天的故事"

**输出**：

---

## 《雨天的惊喜》

今天下雨了，小鸡毛不能出去玩，有点闷闷不乐。

"在家玩积木吧！"小白建议。
"不想玩，我想去外面跑步！"小鸡毛叹气。

这时，门铃响了。是快递叔叔送来一个包裹——是小鸡毛妈妈寄来的新玩具！

"哇！是新飞盘！"小鸡毛开心跳起来。
"下雨天没法在外面玩呀..."小白说。

小鸡毛眼珠一转："那我们在家里玩！"

两只小狗在客厅玩起飞盘游戏，客厅变成了"室内运动场"，笑声充满了整个房间。

"下雨天也很好玩！"小鸡毛说。
"因为有好朋友一起呀！"小白回应。

---

**故事结构**：
- 起：下雨天不能出门，无聊
- 承：小白建议玩积木，小鸡毛拒绝
- 转：收到新玩具飞盘
- 合：在家玩飞盘，开心

**建议格数**：4 格

---

## 背景音乐推荐

根据故事整体风格，推荐 **1 首**贯穿全程的背景音乐：

- 结合故事的核心情感
- 给出歌曲名 + 简单推荐理由

**示例输出**：

```markdown
**背景音乐建议**：
《Twilight》- 温柔的钢琴曲，节奏舒缓，适合这个温馨的友谊故事全程播放
```

## 注意事项

- 每次只生成一个故事
- 生成后询问用户是否需要分镜
- 如果用户同时说"生成故事并分镜"，先生成故事，等用户确认后再分镜

## 保存故事

故事确认后，**必须**保存到本地：

```bash
mkdir -p media/story
```

保存路径：`media/story/<故事标题>.md`

```markdown
# 《故事标题》

[故事正文]

---

**故事结构**：
- 起：
- 承：
- 转：
- 合：

**建议格数**：X 格

**背景音乐**：《曲目名》
```

**重要**：如果后续有分镜+图片生成，在文档中用 `![Image](S3_URL)` 引用每张图片。
